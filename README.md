# Intelligent Process Intelligence & Risk Monitoring System

## Core Concept
A cross-platform system monitoring tool that goes beyond traditional task managers. It continuously observes processes, learns their normal behavior, and identifies anomalies, risks, and system-impacting patterns in real time.

## Key Features

### 1. Intelligent Behavior Analysis
- Detects CPU spikes, sustained high resource usage, and rapid thread creation.
- Identifies memory growth trends (leaks) vs. temporary spikes.
- Distinguishes between normal and abnormal execution patterns.

### 2. Process Lifecycle & Lineage Tracking
- Tracks creation, execution flow, termination.
- Reconstructs parent-child relationships (execution trees).
- Detects orphaned and zombie processes.

### 3. Security & Privilege Awareness
- Evaluates privilege levels (root/high-privilege).
- Highlights unexpected privilege usage.
- Correlates usage with process origin.

### 4. Risk Scoring & Classification
- **Dynamic Risk Score**: Derived from resource usage, privilege, duration, memory patterns, lineage.
- **Classification**: Low, Medium, High, Critical Risk.

### 5. Resource Abuse & System Impact
- Correlates system stress with individual processes.
- Detects sustained abuse vs. brief peaks.

### 6. Event Logging & Timeline
- Logs creation, abnormal behavior, privilege changes, crashes, termination.
- Supports timeline reconstruction with structured logging.

### 7. Incident-Focused Monitoring
- Highlights *changes*: newly spawned processes, privilege escalations, sudden behavior shifts.

## Installation & Usage

### Download (no Rust required)

If you just want to run the app (no compiling), download a prebuilt binary from the GitHub **Releases** page.

1) Open the repository on GitHub → **Releases**
2) Download the asset matching your OS (Windows/Linux/macOS)
3) Run it:

- **Windows**: double-click `process_monitor-...windows-....exe`
- **Linux/macOS**:
	- `chmod +x ./process_monitor-...`
	- `./process_monitor-...`

Note: binaries are generated by GitHub Actions when a tag like `v0.1.0` is pushed.

### Prerequisites
- Rust 1.70+ (install from [rustup.rs](https://rustup.rs))
- Cargo (included with Rust)

### Build & Run

```bash
# Debug build (for development)
cargo build
cargo run

# Release build (optimized)
cargo build --release
cargo run --release
```

### Controls
- **Tab**: Switch between Dashboard, Processes, Graphs, and Help
- **Up/Down**: Scroll through process list (in Processes tab)
- **q**: Quit application

## Architecture

### Modules
- **collector.rs**: System data collection using `sysinfo` crate
- **analyzer.rs**: Behavioral analysis and risk scoring
- **models.rs**: Data structures for processes, metrics, and events
- **app.rs**: Application state and business logic
- **ui.rs**: Terminal UI rendering with `ratatui`
- **logger.rs**: Event logging system
- **main.rs**: Entry point and event loop

### Dependencies
- `sysinfo`: Cross-platform system information
- `ratatui`: Terminal UI framework
- `crossterm`: Terminal manipulation
- `chrono`: Date/time handling
- `env_logger` & `log`: Logging infrastructure

## Platform Support
Builds and runs on Windows, Linux, and macOS.

Cross-platform (all OSes):
- Process list, CPU%, RAM usage, per-process CPU/RAM, best-effort per-process disk I/O deltas
- Disk space (partitions) and basic system info (OS, host, uptime)

Windows-only (via WMI perf counters; shown as N/A on other OSes):
- GPU utilization %, GPU memory (VRAM/shared), per-process GPU%, per-process VRAM
- True PhysicalDisk throughput per disk (MiB/s)

Optional (Windows/Linux with NVIDIA drivers):
- Enable `nvidia-nvml` to get GPU utilization % and VRAM usage via NVML (cross-platform NVIDIA).

Build example:
```bash
cargo run --features nvidia-nvml
```

Note: GPU utilization/VRAM is not exposed through a single cross-platform API. On non-Windows platforms you would need vendor/driver-specific APIs (e.g., NVML/ROCm/Metal tooling), so this app intentionally degrades gracefully instead of failing to compile.
